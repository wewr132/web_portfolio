2. МОДЕЛИ БАЗЫ ДАННЫХ
================================================================================

--------------------------------------------------------------------------------
2.1. users.User (наследуется от AbstractUser)
--------------------------------------------------------------------------------
    Назначение: Кастомная модель пользователя с ролями и расширенным профилем.

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ username        │ CharField     │ Логин (наследуется от AbstractUser)      │
    │ email           │ EmailField    │ Email (наследуется от AbstractUser)      │
    │ first_name      │ CharField     │ Имя (наследуется от AbstractUser)        │
    │ last_name       │ CharField     │ Фамилия (наследуется от AbstractUser)    │
    │ password        │ CharField     │ Пароль (хешируется, наследуется)         │
    │ bio             │ TextField     │ Биография, blank=True                    │
    │ avatar          │ ImageField    │ Аватар, upload_to='avatars/',            │
    │                 │               │ blank=True, null=True                    │
    │ role            │ CharField     │ Роль: ADMIN / MODERATOR / USER,          │
    │                 │               │ default=USER                             │
    │ is_superuser    │ BooleanField  │ Суперпользователь (наследуется)          │
    │ is_staff        │ BooleanField  │ Доступ к админке (наследуется)           │
    │ is_active       │ BooleanField  │ Активен (наследуется)                    │
    │ date_joined     │ DateTimeField │ Дата регистрации (наследуется)           │
    │ last_login      │ DateTimeField │ Последний вход (наследуется)             │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

--------------------------------------------------------------------------------
2.2. gallery.Category
--------------------------------------------------------------------------------
    Назначение: Категории фотографий (например: «Природа», «Студия», «Портреты»).

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ title           │ CharField     │ Название категории, max_length=100       │
    │ slug            │ SlugField     │ URL-имя, unique=True, max_length=100     │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

    Связи: One-to-Many с Photo (category → photos).

--------------------------------------------------------------------------------
2.3. gallery.Photo
--------------------------------------------------------------------------------
    Назначение: Фотографии в портфолио, привязанные к категории.

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ title           │ CharField     │ Название фото, max_length=100            │
    │ img             │ ImageField    │ Файл изображения,                        │
    │                 │               │ upload_to='portfolio_photos/'            │
    │ category        │ ForeignKey    │ Связь с Category, on_delete=CASCADE      │
    │ upload_date     │ DateTimeField │ Дата загрузки, auto_now_add=True         │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

--------------------------------------------------------------------------------
2.4. gallery.SiteInfo
--------------------------------------------------------------------------------
    Назначение: Глобальные настройки сайта (редактируются через админ-панель).

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ slogan          │ CharField     │ Слоган, max_length=255                   │
    │ slogan_bg       │ ImageField    │ Фон слогана (для Hero),                  │
    │                 │               │ upload_to='site/', blank=True, null=True │
    │ about_text      │ TextField     │ Текст «Обо мне»                          │
    │ about_photo     │ ImageField    │ Фото фотографа, upload_to='site/'        │
    │ vk              │ URLField      │ Ссылка на ВКонтакте, blank=True          │
    │ tg              │ URLField      │ Ссылка на Telegram, blank=True           │
    │ inst            │ URLField      │ Ссылка на Instagram, blank=True          │
    │ email           │ EmailField    │ Email для связи, blank=True              │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

    Доступно во всех шаблонах через контекстный процессор site_info.

--------------------------------------------------------------------------------
2.5. services.Order
--------------------------------------------------------------------------------
    Назначение: Заказы на фотосъёмку от пользователей.

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ user            │ ForeignKey    │ Связь с User, on_delete=CASCADE          │
    │ service_type    │ CharField     │ Тип услуги: NATURE / STUDIO / PORTRAITS  │
    │ date            │ DateTimeField │ Дата создания, auto_now_add=True         │
    │ status          │ CharField     │ Статус: NEW / IN_PROGRESS / COMPOLETED,  │
    │                 │               │ default=NEW                              │
    │ contact_info    │ CharField     │ Контактная информация (тел./соц.сети),   │
    │                 │               │ max_length=255                           │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

--------------------------------------------------------------------------------
2.6. services.Review
--------------------------------------------------------------------------------
    Назначение: Отзывы клиентов (публикуются после модерации).

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ user            │ ForeignKey    │ Связь с User, on_delete=CASCADE          │
    │ text            │ TextField     │ Текст отзыва                             │
    │ reting          │ PositiveSmall │ Оценка от 1 до 5                         │
    │                 │ IntegerField  │                                          │
    │ is_published    │ BooleanField  │ Опубликовано, default=False              │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

--------------------------------------------------------------------------------
2.7. chat.ChatMessage
--------------------------------------------------------------------------------
    Назначение: Сообщения в чате с фотографом.

    Поля:
    ┌─────────────────┬───────────────┬──────────────────────────────────────────┐
    │ Поле            │ Тип           │ Описание                                 │
    ├─────────────────┼───────────────┼──────────────────────────────────────────┤
    │ user            │ ForeignKey    │ Связь с User, on_delete=CASCADE          │
    │ text            │ TextField     │ Текст сообщения                          │
    │ created_at      │ DateTimeField │ Время отправки, auto_now_add=True        │
    │ is_deleted      │ BooleanField  │ Помечено как удалённое, default=False    │
    └─────────────────┴───────────────┴──────────────────────────────────────────┘

6. КЛЮЧЕВЫЕ ФУНКЦИОНАЛЬНЫЕ БЛОКИ
================================================================================

6.1. Аутентификация и авторизация
    • Кастомная модель User с ролями (ADMIN, MODERATOR, USER)
    • Регистрация с авто-логином (redirect на главную)
    • Вход / Выход / Смена пароля
    • LoginRequiredMixin на всех контроллерах кроме главной, портфолио и отзывов

6.2. Профиль пользователя
    • Редактирование: username, email, first_name, last_name, bio, avatar
    • Отображение аватарки (или заглушка с первой буквой имени)
    • Кнопка «Сменить пароль» в профиле

6.3. Главная страница (Landing Page)
    • Hero-блок (100vh) с параллакс-фоном (загружается через админку)
    • Navbar фиксированный поверх Hero
    • Сетка последних 6 фотографий (grid, 1-3 колонки)
    • Блок «Обо мне» (Flexbox 55/45, фото + текст)
    • CTA-блок «Связаться со мной» (тёмный фон, ghost button → чат)

6.4. Портфолио
    • Pill-навигация по категориям (HTMX, без перезагрузки)
    • Masonry-сетка фотографий (CSS columns, адаптив 1-4 колонки)
    • Фильтрация по категории

6.5. Заказы (Services)
    • Создание заказа (тип услуги из выпадающего списка, контактная информация)
    • Список заказов пользователя (только свои)
    • Статусы: Новый → В работе → Завершён

6.6. Отзывы
    • Публичный список (только опубликованные, is_published=True)
    • Создание отзыва (только авторизованные, уходит на модерацию)
    • Card UI с аватаркой, именем, рейтингом ★/☆
    • Контактные данные фотографа в блоке «Обо мне»

6.7. Чат
    • Только для авторизованных
    • История сообщений (последние внизу, автопрокрутка)
    • Отправка в реальном времени (с перезагрузкой страницы, без WebSockets)

6.8. Админ-панель
    • Управление пользователями
    • Управление категориями и фотографиями
    • Настройки сайта (SiteInfo: слоган, фон, обо мне, контакты)
    • Заказы (с отображением email пользователя)
    • Отзывы (модерация: is_published)

6.9. UX-улучшения
    • Sticky footer (body flex-column, footer margin-top: auto)
    • Toast-уведомления (фиксированные, закрываются кликом)
    • LOGIN_URL = '/users/login/' для миксинов
    • Локализация: ru-ru, Europe/Moscow
    • Контекстный процессор site_info доступен во всех шаблонах
    • Авто-логин после регистрации
